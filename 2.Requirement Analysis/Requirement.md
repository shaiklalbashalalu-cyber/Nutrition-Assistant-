# Phase 2: Requirement Analysis 📋

This folder outlines the specific functional and non-functional requirements needed to build the Nutrition Assistant.

## Functional Requirements
* User authentication (Sign up, Log in, Log out).
* Search functionality for the food database.
* Daily calorie and macro calculation logic.
* User dashboard to display nutritional progress.

## Non-Functional Requirements
* **Performance:** API responses must be under 200ms.
* **Responsiveness:** The UI must work perfectly on both mobile and desktop screens.
* **Security:** User data and passwords must be encrypted.

## User Stories
* *As a user, I want to search for an apple so I can log its calories.*
* *As a user, I want to see a daily summary chart so I can track my macros.*
